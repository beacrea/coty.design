Yes — the strongest pattern emerging is not a secret “bot-only” website, but a public machine-readable layer beside your normal pages: entity schema, stable canonical profile pages, clean markdown or JSON endpoints, optional `/llms.txt`, and granular crawler rules. The tactics with the clearest near-term payoff are structured data and fetchable clean pages, while `/llms.txt` is promising but still early, and agent detection is best used to change **format**, not to hide materially different claims.[1][2][3][4][5][6][7]

## Conventions

The biggest convention shift is toward entity-first publishing: sites are marking up people, organizations, articles, FAQs, and other entities so AI systems can identify who someone is, what they know, and how claims connect across the web. For a personal site, that means your About page is no longer just marketing copy; it is a canonical identity record that should reinforce your expertise, affiliations, and corroborating links.[3][4]

A second convention is crawler-role separation rather than one blanket bot policy. OpenAI documents separate crawlers for different functions, and Anthropic now breaks out ClaudeBot, Claude-User, and Claude-SearchBot with distinct roles, which makes “allow search and retrieval, restrict training” a more practical policy stance than simple allow/deny rules.[5][8][9]

A third convention is publishing LLM-friendly summaries and source maps. The `/llms.txt` proposal defines a root markdown file with a short summary plus links to deeper markdown resources, and its own description explicitly calls out personal portfolio and CV sites as a use case.[6][7]

## Strategy fit

| Strategy | Current convention | Efficacy now |
|---|---|---|
| JSON-LD entity layer | Use `Person` plus related schema such as organization/article/FAQ where appropriate, and keep it consistent across pages [3][4] | High: this helps systems verify identity, expertise, and brand/entity relationships [3][4] |
| Clean dossier pages | Publish stable HTML pages with clean markdown or structured mirrors that agents can fetch and parse easily [6][7] | High: this aligns with the inference-time retrieval model described by the `/llms.txt` proposal [6][7] |
| Bot-specific crawl policy | Set separate rules for training bots, search bots, and user-initiated retrieval bots [5][9][8] | Medium: useful for steering access, but not a hard guarantee because crawler compliance is uneven [10][5][9] |
| `/llms.txt` | Add a root summary file that links to deeper machine-friendly resources [6][7] | Experimental: one write-up citing a 300k-domain analysis reported 10.13% adoption, while a separate top-100 tracker showed 0% among the biggest sites [1][2] |

## Implementation

For coty.design, I would treat the agent layer as a public dossier surface rather than a hidden alternate reality. Concretely, that means stable pages for `/about`, `/bio-short`, `/bio-long`, `/speaking`, `/work`, `/projects`, `/timeline`, and `/contact`, with the same core facts expressed in HTML for humans, JSON-LD in-page for parsers, and markdown or JSON mirrors for retrieval agents.[7][3][6]

The content itself should be highly answerable. Put your role definitions, domain areas, differentiators, notable outcomes, speaking topics, preferred collaboration types, geography, and proof links into tight factual blocks so an agent assembling a hiring or speaker brief has fewer chances to improvise. A good mental model is “design a canonical packet that a research assistant could quote from safely.”

## Guardrails

Agent detection still matters, but mainly as a rendering switch. It is reasonable to serve a denser version to likely agents — more headings, facts, citations, structured lists, last-updated dates, and explicit source links — while keeping the underlying claims aligned with the human page.

I would avoid making the agent response materially different in substance from the human response, because that creates trust and discoverability risk. That caution matters more because the bot landscape is fragmented across search, training, and user-retrieval agents, and reporting has also alleged that some AI crawlers have bypassed `robots.txt`, which means exclusion controls are signals, not perfect enforcement.[9][10][5]
